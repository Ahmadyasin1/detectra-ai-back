All layers communicate via async message queues to achieve non-blocking, scalable throughput. Layer 1 · Ingestion RTSP stream reader, video file uploader, frame extractor, audio splitter. Entry point for all video data. Layer 2 · Preprocessing Frame normalization, resizing, denoising, optical flow, audio resampling & spectrogram generation. Layer 3 · Perception Object detection, logo recognition, face/pose detection, action recognition, speech-to-text, environmental audio. Layer 4 · Reasoning Multimodal fusion engine (transformer), temporal event sequencing, context inference, story/scene understanding. Layer 5 · Storage PostgreSQL for metadata, Redis for live event stream, S3-compatible blob for video/thumbnails, TimescaleDB for time-series events. Layer 6 · API + UI FastAPI REST + WebSocket gateway, Next.js dashboard, real-time timeline, alert engine, report export. OpenCV · FFmpeg · asyncio FrameRouter Dispatches frames to parallel GPU workers. Load balances across available CUDA devices. Drops frames if queue saturated (priority: keyframes). import cv2, asyncio, time

class RTSPStream:
    def __init__(self, url: str, fps: int = 5):
        self.url = url
        self.fps = fps
        self.cap = None

    async def connect(self):
        self.cap = cv2.VideoCapture(self.url, cv2.CAP_FFMPEG)
        self.cap.set(cv2.CAP_PROP_BUFFERSIZE, 3)
        self.cap.set(cv2.CAP_PROP_FPS, self.fps)

    async def read_frames(self):
        interval = 1.0 / self.fps
        while True:
            ret, frame = self.cap.read()
            if not ret:
                await asyncio.sleep(2)
                await self.connect()  # auto-reconnect
                continue
            yield frame, time.time_ns() // 1_000_000  # timestamp_ms
            await asyncio.sleep(interval) Edge vs Cloud Architecture Edge Node (On-Site) StreamAgent, PerceptionAgent (YOLOv8, ByteTrack, Pose). Runs on local GPU server or Jetson AGX Orin. Multiple models competing on shared GPU causes OOM crashes. Mitigation: Run Mistral-7B on CPU (acceptable at 30s processing cycle). Use model offloading — load SlowFast only when a person is detected (lazy loading). Allocate Whisper to CPU with whisper.cpp. Profile with nvidia-smi during load testing. RTSP Stream Instability HIGH IP cameras drop streams under network stress. OpenCV's VideoCapture silently fails — ret=False without exception. Unrecovered drops stall the entire pipeline. Mitigation: Implement exponential backoff reconnect logic (1s, 2s, 4s, 8s). Heartbeat check every 10s — if no frame received, trigger reconnect. Dashboard shows camera status (connected / reconnecting / offline). Real-World Domain Gap HIGH Models trained on benchmark datasets (COCO, HMDB51) under ideal conditions perform poorly on actual surveillance footage: low resolution, motion blur, lens distortion, extreme angles, poor lighting. Mitigation: Domain adaptation fine-tuning on 500 custom IP camera frames. Test-time augmentation (TTA) with horizontal flip averaging. CLAHE preprocessing for low-light. Plan 10% of total project time for domain-specific model tuning. LLM Hallucination MEDIUM Mistral-7B may generate inaccurate scene narratives or incorrect anomaly classifications, especially for ambiguous events. False alert generation undermines system trust. Mitigation: Constrain LLM output with JSON schema validation. Use few-shot prompting with 5 labeled examples per scene type. Never trigger critical alerts from LLM alone — always require PerceptionAgent confirmation. Display confidence score and "AI-generated" label on all narratives. AV Synchronization Drift MEDIUM Audio and video streams may drift over time (common in RTSP). Misaligned timestamps cause fusion engine to correlate audio events with wrong video frames. Mitigation: Use PTS (Presentation Timestamp) from FFmpeg — do not rely on wall clock. Implement NTP sync check at stream start. Add drift monitor: if A/V offset >200ms, flag and trigger re-sync. Use FFmpeg's -use_wallclock_as_timestamps 1 for live streams. Privacy + Ethical Risk MEDIUM System processes identifiable human data (faces, movements, speech). Facial recognition, if added, may violate Pakistani data privacy norms. Misuse for unauthorized surveillance. Mitigation: Do NOT implement facial recognition in Phase 1 — use anonymous tracking IDs only. All stored video footage encrypted at rest (AES-256). Access control: JWT auth required for all API endpoints. Academic use declaration in FYP documentation. No real personal data used in demos. Limited GPU Hardware MEDIUM UCP lab GPUs may have limited availability. Training and running all models on a single consumer GPU is challenging at scale. Mitigation: Use Google Colab Pro+ (A100) for training. Download pre-trained weights for most models (no training needed).
