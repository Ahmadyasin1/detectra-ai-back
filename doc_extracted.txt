Unlike existing systems that rely on third-party APIs, Detectra AI integrates object detection, logo recognition, motion/activity recognition, and audio transcription (including environmental sounds and speech) into a single platform. Using a transformer-based fusion engine, the system synchronizes outputs from multiple modalities into a unified, timeline-based dashboard, enabling deeper contextual understanding and real-time analysis. This project addresses key challenges in current video analysis tools, including limitations in contextual integration, privacy concerns, and scalability issues associated with API dependencies. By offering a self-contained, API-free framework, Detectra AI ensures enhanced privacy, customization, and scalability, making it ideal for applications in surveillance, sports analytics, media monitoring, and other domains. The project covers a broad range of technical knowledge, including deep learning (CNNs, Transformers), computer vision (YOLO, DETR, Vision Transformers), natural language processing (speech-to-text), and web development (Flask, Next.js), culminating in an interactive, timeline-based dashboard that delivers actionable insights across various industries.IntroductionProductExisting video analysis systems rely heavily on third-party APIs, which limits customization and raises privacy concerns. These systems process different modalities (visual, audio, text) separately, failing to capture holistic context and lacking unified reporting mechanisms. The core problem is the lack of a self-sufficient, multimodal video intelligence platform.BackgroundVideo data growth has been exponential, but analysis tools are fragmented, often requiring separate APIs for visual objects, audio cues, or text. This modularity hinders real-time contextual understanding and creates dependence on external services, leading to issues in data privacy, scalability, and integration. Recent multimodal AI advancements (e.g., CLIP, GPT-4V) show promise but often lack deployment flexibility. Detectra AI is designed to address this by offering a holistic, integrated framework that processes video, audio, and language modalities together in a self-contained manner.Objective(s)/Aim(s)/Target(s)To build a web interface using React and FastAPI for local video processing.To deploy YOLOv11 and Vision Transformers for object and logo detection.To utilize MediaPipe and ST-GCN for human motion and action recognition.To implement Whisper and PANNs for local speech-to-text and audio analysis.To develop a Transformer fusion engine to contextually align audio and video data.To design an interactive dashboard for timeline-based reports and visual analytics.To ensure total autonomy by hosting all models locally via Docker containers.ScopeThe scope of this SRS covers the development of a unified web application called Detectra AI for end-to-end multimodal video analysis. This includes all required modules for object detection, logo recognition, motion/activity recognition, audio transcription, a transformer-based fusion engine, and a final interactive, timeline-based dashboard.Business GoalsThe software addresses the corporate goal of providing an API-free, privacy-preserving video analysis tool and has industry relevance for surveillance, sports analytics, media monitoring, and smart cities.Document ConventionsItalicized text represents project-specific technical terms (e.g., Multimodal Fusion Engine). Acronyms (e.g., SRS, AI, DETR) are defined in Appendix A.MiscellaneousThe project emphasizes ethical AI practices, including bias mitigation in model training. Team collaboration will use tools like Git for version control and Slack for communication. It processes data in a pipeline: video frames are analyzed for visuals, audio is transcribed, and outputs are fused into a timeline. This extends the functional requirements from Phase 1, such as use cases for uploading, analyzing, and viewing reports.User Classes and CharacteristicsPrimary users are analysts and researchers in surveillance, sports, media, and journalism. They possess technical knowledge of video data, value privacy and autonomy, and require time-stamped, actionable insights. Secondary users may include administrators for system maintenance.Design and Implementation ConstraintsDevelopment must use PyTorch/TensorFlow for AI models, Hugging Face for pre-trained components, OpenCV and Librosa for preprocessing, and Flask/Next.js for web frameworks. Hardware is limited to GPU-enabled workstations, with optimizations for inference time. Compliance with data privacy regulations is mandatory, and no external APIs are permitted.Assumptions and DependenciesAssumptions include access to GPU hardware and sufficient open-source datasets (e.g., COCO, AudioSet). Dependencies involve stable libraries like YOLO, DETR, Whisper, and FFmpeg. Changes in library support could impact timelines. Technical ArchitectureDetectra AI adopts a layered architecture with client-server separation, ensuring modularity for maintenance and scalability. The system is custom-built for batch processing of uploaded videos, focusing on transaction-based analysis rather than real-time streaming.The system is entirely custom-built using open-source libraries and frameworks, with no reliance on Commercial Off-The-Shelf (COTS) software beyond foundational tools like Python libraries. It handles batch processing for video uploads and analysis, supporting transaction processing for user-initiated tasks (e.g., video upload and report generation) and analytical reporting via the timeline dashboard. There is no real-time online processing; all operations are triggered on-demand.Major application components include:Frontend: User interface for uploads, configurations, and viewing results.Backend: Handles API endpoints, video processing orchestration, and data flow.AI Inference Engine: Modular components for visual and audio analysis.Fusion Engine: Integrates multimodal outputs.Storage Layer: Temporary file handling for inputs/outputs.The system collects and manages video files, extracted frames/audio, model outputs (e.g., JSON detections, transcripts), and fused timelines. No persistent user data is stored beyond session-based results.The basic application architecture is layered client-server:Client (browser-based frontend) communicates via REST APIs to the server.Server (backend) orchestrates processing and returns results.The system is built primarily in Python (backend and AI modules), with JavaScript/TypeScript for the frontend (Next.js). It runs on hardware platforms supporting GPU acceleration, such as NVIDIA CUDA-enabled workstations or cloud VMs (e.g., AWS EC2 with GPU). The database platform is minimal using in-memory storage (e.g., Redis for sessions) and file-based storage (e.g., local filesystem or S3-compatible for videos); no relational database like SQL is used for simplicity.The end-user interface is browser-based (thin client), accessible via modern web browsers (Chrome, Firefox, etc.). No thick client is required.The basic network architecture supports LAN, WAN, and Internet access, with HTTPS for secure communication. The system is hosted locally on university servers or an external cloud data center (e.g., AWS or Google Cloud) for development and demo purposes; it is not hosted in a specific CMS or enterprise data center.High-Level Architecture Diagram (Textual Representation using ASCII Art): (3.0.1) Subsytem Architecture Application and Data ArchitectureThe architecture comprises:Frontend Layer: Built with Next.js, handling user interfaces and API calls.Backend Layer: Flask-based server managing video uploads, AI inference, and fusion.AI Modules: Independent components for object detection (YOLO/DETR), logo recognition (custom ViT), motion recognition (Anticipative Video Transformer), and audio analysis (wav2vec 2.0/Whisper).Fusion Engine: Transformer model aligning modalities via embeddings.Data Storage: Temporary file storage for videos and JSON outputs; no persistent database for simplicity.(3.1.2)Architecture Diagram (UML Based) (3.1.3) Multimodal Analysis Pipeline(3.1.4)UML Class Diagram(3.1.5)UML Component Diagram(3.1.6) Entity Relationship DiagramComponent Interactions and Collaborations(3.2.1) Activity Diagram(3.2.2) Event Traces Diagram(3.2.3) Sequence Diagram	         (3.2.4) UML Collaboration Diagram(3.2.5) DFD Diagram(3.2.6) User Case DiagramDesign Reuse and Design PatternsReuse includes pre-trained models from Hugging Face (e.g., DETR for detection). Patterns: Singleton for Fusion Engine to ensure one instance; Observer for real-time progress updates; Factory for creating analysis modules dynamically.Technology ArchitectureThe infrastructure for Detectra AI is designed to support high-compute multimodal processing while maintaining a lightweight, accessible user interface. The architecture follows a client-server model that prioritizes modularity and GPU-accelerated performance.Platform and HostingThe system is built on a cross-platform Python-based backend capable of running on Linux, Windows, or macOS. To handle the intensive requirements of deep learning models like YOLO and Vision Transformers, the hosting environment requires GPU-enabled workstations or cloud-based Virtual Machines (e.g., AWS EC2 or Google Cloud) equipped with NVIDIA CUDA-enabled hardware.Connectivity and Network RequirementsAccess Protocol: The system is accessed via standard HTTP/HTTPS protocols through modern web browsers.Network Environment: The architecture supports deployment across LAN, WAN, and the Internet, making it flexible for both local university server hosting and broader cloud deployment.Client Type: It utilizes a thin-client approach (Next.js), ensuring that users do not need to install specialized hardware or software locally to view reports.Modes of OperationsBatch Processing: The system is optimized for transaction-based batch processing rather than real-time streaming. Users initiate an analysis task by uploading a video, which the backend then processes through the sequential AI pipeline.On-Demand Execution: All operations including visual detection, audio transcription, and multimodal fusion are triggered on-demand by the user.Information & Data InfrastructureStorage: The infrastructure utilizes file-based storage (local filesystem or S3-compatible) for video assets and in-memory storage (e.g., Redis) for session management.Database: To maintain simplicity and privacy, no persistent relational SQL database is implemented; all analysis outputs are delivered as session-based JSON results.Screenshots/Prototype WorkflowSwim-lane Diagram: Lanes for User, Frontend, Backend, AI Engine.
